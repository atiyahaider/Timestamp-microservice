API Project: Timestamp Microservice

    User Stories :

  1. The API endpoint is GET https://vast-butterfly.glitch.me/api/timestamp/:date_string?
  2. A date string is valid if can be successfully parsed by new Date(date_string).
    The unix timestamp should be an integer (not a string) specifying milliseconds.
    Date strings should be compliant with ISO-8601 (e.g. "2019-01-13") because this will ensure a UTC timestamp.
  3. If the date string is empty it would be equivalent to new Date(), i.e. the service uses the current timestamp.
  4. If the date string is valid the API returns a JSON having the structure
           {"unix": <date.getTime()>, "utc": <date.toUTCString()> }
    e.g. {"unix": 1547387904000, "utc": "Sun, 13 Jan 2019 13:58:24 GMT"}
  5. If the date string is invalid the API returns a JSON having the structure
    {"unix": null, "utc": "Invalid Date" }.

Example Usage:

  • https://vast-butterfly.glitch.me/api/timestamp/2019-01-13
  • https://vast-butterfly.glitch.me/api/timestamp/1547337600000

Example Output:

{"unix":1547337600000, "utc":"Sun, 13 Jan 2019 00:00:00 GMT"}
